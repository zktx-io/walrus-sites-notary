# Support Policy

This project documents support expectations for each release as follows.

## Support Scope

- The latest tagged release is the only release that is fully supported for functional fixes and security updates.
- The `main` branch represents ongoing development and is not a supported release artifact.

## Support Duration

- A release remains fully supported until a newer tagged release is published.
- Superseded releases may receive best-effort fixes for up to 30 days after a newer release, but this is not guaranteed.

## End of Security Updates

- When a release is superseded and the 30-day best-effort period has ended, it should be considered out of support for security updates.
- Release notes may define a shorter or longer exception window when operationally necessary.

## Documentation Requirement

Each official release should state whether it is:

- Current and fully supported
- Superseded and best-effort only
- End of support
